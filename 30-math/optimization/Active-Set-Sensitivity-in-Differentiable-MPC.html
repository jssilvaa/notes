Active Set Sensitivity in Differentiable MPC

Active-Set Sensitivity in Differentiable MPC

May 19, 20267 min read

  • robotics/control
  • math/optimization
  • blog-candidate

Summary

Differentiating through a parametric QP (as in OptNet / MPC.pytorch) requires ∂z⋆/∂θ, which exists and is C1 only while the active set A(θ) is constant. The derivation applies the implicit function theorem to the reduced KKT system; the resulting linear solve gives the Jacobian without re-solving the QP. At active-set boundaries the gradient is set-valued (Clarke subdifferential), and any autodiff library silently picks one element. Empirically, two failure modes show up in even the smallest bilevel toy problem: (i) silent disagreement with finite differences at degenerate KKT points, and (ii) outer-loop convergence to a projected — not global — optimum when the inner Jacobian becomes rank-deficient on an active manifold.

Core Mechanism

Parametric QP.

z⋆(θ)=argzmin​21​z⊤Q(θ)z+q(θ)⊤zs.t.A(θ)z=b(θ),G(θ)z≤h(θ)

KKT residual (active constraints only, under strict complementarity).

F(z,ν,λA​;θ)=​Qz+q+A⊤ν+GA⊤​λA​Az−bGA​z−hA​​​=0

IFT application. Since F(z⋆,ν⋆,λA⋆​;θ0​)=0, the IFT gives

∂θ∂(z⋆,ν⋆,λA⋆​)​=−K−1∂θ∂F​,K=​QAGA​​A⊤00​GA⊤​00​​

K is invertible iff LICQ holds ([A;GA​] has full row rank — guarantees unique multipliers) and SOSC holds (d⊤Qd>0 for all d in the critical cone — guarantees z⋆ is an isolated minimiser). Proof sketch: Kv=0⇒dz​ lies in the critical cone SOSC​dz​=0LICQ​dν​=dλ​=0.

Coding loop (challenge.py). For the outer bilevel problem minq​21​∥z⋆(q)−ztgt​∥2:

  1. Solve inner QP → get z⋆, λ⋆.
  2. Detect A={i:λi⋆​>ϵ} (used ϵ=10−6).
  3. Assemble K, solve K[∂z⋆;∂λA​]⊤=−[I;0] for ∂z⋆/∂q.
  4. Outer gradient: ∇q​ℓ=(∂z⋆/∂q)⊤(z⋆−ztgt​).
  5. Update q←q−η∇q​ℓ (used η=0.1, 200 iterations).

At boundaries — Clarke subdifferential. For locally Lipschitz z⋆ (Robinson’s theorem guarantees this; Rademacher’s theorem then ensures the limit Jacobians in the definition below exist a.e., so the set is non-empty):

∂Cz⋆(θ)=conv{klim​∇z⋆(θk​):θk​→θ,z⋆ differentiable at θk​}

Box example: z⋆(θ)=clip(θ,−1,1), ∂Cz⋆(1)=[0,1]. Autodiff picks one point silently.

Results

Ran challenge.py (problem: z∈R2, Q=I, constraints z1​+z2​≤1, z≥0, target ztgt​=(0.3,0.6)). Three regimes tested.

(a) Interior, well-behaved active region — q=(−1,−1). Unconstrained optimum −q=(1,1) violates z1​+z2​≤1, so constraint 0 binds: A={0}, z⋆=(0.5,0.5). Analytic and finite-difference Jacobians agree to 3×10−12:

∂z⋆/∂q=[−0.50.5​0.5−0.5​]

This is the IFT working exactly as advertised — and notice the determinant is zero (foreshadowing the rank-deficiency result below).

(b) Degenerate KKT, silent failure — q=(−0.5,−0.5). Unconstrained optimum z⋆=(0.5,0.5) lies exactly on z1​+z2​=1, but the solver returns λ=0. Strict complementarity fails. With the active-set threshold ϵ=10−6, the code classifies this as A=∅ and returns analytic Jacobian −I. Central differences return

[−0.750.25​0.25−0.75​]=21​(−I)+21​[−0.50.5​0.5−0.5​]

— exactly the mean of the two adjacent regime Jacobians, a Clarke subgradient element. The disagreement is O(1), not O(ϵ); no NaN, no warning, no convergence failure. The two-dimensional analogue of the box example from the deck, observed in 30 lines of Python.

(c) Rank-deficient Jacobian, stuck optimisation — q0​=(−1,−2). Initialised on the active manifold z1​+z2​=1. Active set stays {0} for all 200 iterations. Final state:

  • qfinal​=(−1.35,−1.65), zfinal⋆​=(0.35,0.65)
  • final loss =2.5×10−3 (not zero), target ztgt​=(0.3,0.6)

The outer Jacobian ∂z⋆/∂q has rank 1 on this active set (determinant zero, null space spanned by (1,1) — the direction that would deactivate the constraint). Gradient descent literally cannot see that direction. The optimisation converges to the projection of ztgt​ onto the active manifold {z:z1​+z2​=1}, not to ztgt​ itself. A correct solution requires either a larger step (overshoots the constraint), regularisation that pushes q into a different regime, or an interior-point smoothing of the inner problem.

Why It Matters

In humanoid centroidal MPC, contact forces hit friction-cone boundaries and joint torques saturate mid-rollout. The active set A changes exactly then. Any system-identification or cost-learning loop that backprops through the MPC (e.g., fitting Q, q from motion capture) will receive a spurious gradient at those transitions — not NaN, just wrong. Knowing which iteration the transition occurred (visible from the dual λ⋆ going to zero) tells you whether to trust the analytic Jacobian or fall back to finite differences for that step.

The result (c) above is the more insidious failure mode: when the outer loop runs entirely inside one active regime, the gradient is perfectly correct locally but the global problem is unsolvable by gradient descent. In MPC training this looks like a training curve that plateaus mid-loss with no sign of trouble.

Separately: the sensitivity warm-start trick zk+1⋆​≈zk⋆​+(∂z⋆/∂θ)Δθ (Diehl & Bock) replaces a full QP solve with one linear solve — practical for MPC at ≥1 kHz.

Limitations

  • Active-set changes invalidate the Jacobian. The IFT is local; K−1 is only valid while A is constant. Transitions happen on a measure-zero set of θ, but they concentrate exactly in the dynamically interesting regimes (contact events, saturation).
  • Degenerate KKT (LICQ failure). Redundant active constraints (e.g., both feet + hand fully constraining the CoM) make [A;GA​] rank-deficient; K is singular. Pirnay et al. 2012 handle this for IPOPT; the analogous SOCP result (friction cones) is an open question.
  • Strict complementarity assumption. At λi⋆​=0 with Gi​z⋆=hi​ (degenerate active constraint), the inactive-multiplier argument breaks down and ∂θ​λi⋆​=0 in general. Empirically observed in result (b) above — the analytic Jacobian and FD disagree by O(1) with no warning.
  • Rank-deficient outer Jacobian on active sets. When constraints bind, ∂z⋆/∂θ lives in the tangent space of the constraint manifold (rank =n−∣A∣). Outer gradient descent cannot move z⋆ off the active set; bilevel loops can converge to manifold-projected, not globally optimal, solutions. Empirically observed in result (c). Mitigation: regularise θ to stay away from boundaries, use multi-start, or smooth the inner problem (interior-point).
  • Smoothing bias. Log-barrier workarounds (OptNet’s approach) give C∞ gradients but introduce bias O(1/t) between the barrier solution zt​ and the true z⋆.

Open Questions

  • For the centroidal MPC push-recovery case: which constraint (∣fx​∣≤μfz​ vs. joint torque limit) is active most often? Could you precompute a stratified K−1 per expected active-set “patch” and cache it?
  • Regularising dual variables to avoid transitions (penalise ∥λ∥ in the outer loss) reduces gradient variance in PPO but costs optimality — what is the Pareto frontier empirically?
  • Does the SOCP friction cone admit the same parametric sensitivity theory as the LP/QP case? (Pirnay et al. 2012 covers NLP; SOCP structure may allow a cleaner closed form via cone projection Jacobians.)
  • The rank-deficiency stuck-optimum failure in result (c) suggests a diagnostic: monitor rank(∂z⋆/∂θ) during training and alert when it drops. Is this a known technique in the differentiable-optimisation literature, or worth writing up?
